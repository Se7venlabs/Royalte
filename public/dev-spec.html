Section 01

What we're building

Royaltē is a music audit and intelligence engine. An artist pastes a Spotify or Apple Music URL and within 60 seconds receives a scored audit report showing metadata gaps, platform coverage issues, publishing risk indicators, duplicate detection flags, YouTube/UGC opportunities, and sync readiness scores.

Critical constraint: Royaltē is NOT a royalty collection platform, NOT a distributor, and NOT a PRO. We surface likely risks and opportunities based on publicly available metadata only. We do not access private royalty statements, private earnings data, or rights-chain records. All findings must be framed as "possible", "likely", or "flagged for investigation" — never as confirmed missing money.

Section 05

The 6 detection modules

Module A — Metadata Integrity

Cross-references artist name, track title, album, release date, ISRC, UPC, explicit flag, and contributor data across Spotify, Apple Music, and MusicBrainz. Flags mismatches and missing identifiers.

// Key rules
IF isrc == null across all sources          → flag("Missing ISRC", severity="high")
IF upc == null                              → flag("Missing UPC", severity="medium")
IF contributors.length == 0                 → flag("No songwriter data", severity="high")
IF spotify.title != apple.title             → flag("Title mismatch", severity="medium")
IF spotify.releaseDate != musicbrainz.date  → flag("Release date conflict", severity="low")
IF spotify.explicit != apple.explicit       → flag("Explicit flag conflict", severity="low")

Module B — Platform Coverage

Checks whether the track/artist is found on Spotify, Apple Music, YouTube, MusicBrainz, and Tidal. Returns found/not-found with confidence score per platform.

IF spotify.found == true AND apple.found == false  → flag("Apple Music gap", severity="medium")
IF youtube.officialPresence == false               → flag("No official YouTube presence", severity="medium")
IF musicbrainz.found == false                      → flag("Not in MusicBrainz", severity="low")
IF coverageScore < 3 of 5 platforms               → flag("Limited platform coverage", severity="high")

Module C — Territory Coverage Audit

Cross-references stream presence by territory against detected PRO registration signals. Flags territories with significant streams but no matching registration. Uses public metadata only — not private PRO records.

IF territory.streams > 100000 AND territory.proStatus == "not_found"
  → flag("Possible PRO gap in " + territory.name, severity="high")

IF territory.streams > 50000 AND territory.proStatus == "pending"
  → flag("Registration pending in " + territory.name, severity="medium")

// Key territories to always check:
// DE=GEMA, FR=SACEM, BR=ECAD, JP=JASRAC, SE=STIM, NO=TONO, DK=KODA, NL=BUMA

Module D — Publishing Risk

Scores publishing risk based on absence of songwriter data, identifier conflicts, metadata weakness, and fragmentation signals. Cannot verify actual unpaid royalties.

IF contributors.songwriters == 0            → publishingRisk += 40
IF isrc == null                             → publishingRisk += 25
IF metadata conflicts > 2                   → publishingRisk += 20
IF label == "self-released" AND isrc == null → publishingRisk += 15
// Score: 0-30 = Low, 31-60 = Medium, 61+ = High

Module E — Duplicate & Fragmentation Detection

IF titleSimilarity > 0.90 AND artistSimilarity > 0.95 AND IDs conflict
  → flag("Possible duplicate recording", severity="medium")
IF same artist has 3+ near-identical titles with different ISRCs
  → flag("Possible catalog fragmentation", severity="medium")

Module F — YouTube / UGC Detection

IF officialAudio == false AND officialVideo == false
  → flag("No official YouTube presence", severity="medium")
IF ugcResults > 10 AND contentIdActive == false
  → flag("UGC detected — Content ID not active", severity="high")

Module G — Sync Readiness

syncScore = 0
IF isrc != null         → syncScore += 20
IF artwork != null      → syncScore += 15
IF allPlatforms >= 4    → syncScore += 20
IF titleFormatClean     → syncScore += 15
IF genreTags.length > 0 → syncScore += 15
IF moodTags.length > 0  → syncScore += 15
// Score out of 100. Below 50 = flag sync readiness gaps
Section 06

Backend flow — paste to report

// Full sequence from URL paste to report returned

1. POST /api/audit/start
   └─ Receive URL
   └─ Parse URL → detect platform + type + extract ID
   └─ Return: { auditId, platform, type, sourceId }

2. POST /api/audit/resolve/:auditId
   └─ Call Spotify API (or Apple Music API)
   └─ Normalise to internal object
   └─ Save to resolved_tracks / resolved_artists table

3. POST /api/audit/run/:auditId
   └─ Run Module A (Metadata) — Spotify + Apple + MusicBrainz
   └─ Run Module B (Platform Coverage) — all platforms in parallel
   └─ Run Module C (Territory Coverage) — public stream + PRO signals
   └─ Run Module D (Publishing Risk) — rule engine scoring
   └─ Run Module E (Duplicates) — MusicBrainz + Spotify search
   └─ Run Module F (YouTube/UGC) — YouTube Data API
   └─ Run Module G (Sync Readiness) — scoring rules
   └─ Save all flags to audit_flags table
   └─ Calculate scores → save to audit_scores table

4. // FREE PREVIEW — returns here, NO Claude call
   GET /api/audit/preview/:auditId
   └─ Return: { score, topFlags[0..1], platform }

5. // PAID FULL REPORT — only after Stripe webhook confirms payment
   POST /api/audit/unlock/:auditId
   └─ Verify Stripe payment confirmed
   └─ Call Claude API with full flag data
   └─ Claude writes plain-English explanations + recommendations
   └─ Save full report to audits table
   └─ Return: { fullReport, allFlags, allScores, actions, territories }

Section 10

Honest MVP limitations

What V1 can detect: Metadata inconsistencies across public sources. Missing ISRCs and UPCs. Platform coverage gaps. Territory stream presence vs PRO registration signals. Publishing risk indicators based on missing data. YouTube official presence. Basic duplicate detection by title similarity. Sync readiness based on metadata completeness.

What V1 cannot verify: Private royalty statements. Confirmed unpaid publishing royalties. Actual earnings by territory. YouTube Content ID fingerprint matching (requires Content ID partner program access). Rights-chain ownership data. Distributor payout history. PRO registration confirmation (inferred only from public signals).

All findings must be communicated as "possible", "likely", or "flagged for investigation" — never as confirmed missing money or verified unpaid royalties. This is both a legal requirement and a product positioning requirement.
