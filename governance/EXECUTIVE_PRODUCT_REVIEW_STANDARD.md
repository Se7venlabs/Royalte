# Royaltē Executive Product Review Standard™

**Authority:** Board-mandated. Mandatory for all current and future Executive Workspaces.
**Status:** ACTIVE — 2026-07-04
**Supersedes:** Nothing — this is a new mandatory gate, not a revision.

---

## The Gate

A workspace is not permitted to merge into main until all three gates are cleared:

| Gate | Requirement |
|------|-------------|
| ✓ Development Complete | All planned features implemented and committed |
| ✓ Product Review Complete | Full Executive Product Review™ conducted per this standard |
| ✓ Board Review Complete | Board has reviewed Vercel Preview and approved |

Feature complete **≠** product complete. The Product Review is not optional.

---

## Scope

This standard applies to every Executive Workspace, without exception:

- Health Intelligence™
- Identity Intelligence™
- Publishing Intelligence™
- Catalog Intelligence™
- Global Music Footprint™
- Backend Intelligence™
- Change Detection™
- AI Insights™
- Any future Executive Workspace

---

## The Review Process

After development is complete:

**STOP.** Do not begin the next workspace.

Open the completed workspace. Review it as though you are an artist using Royaltē for the very first time.

**Do not review the code. Review the experience.**

Ask one question throughout the entire review:

> *"Does this still feel like one operating system?"*

---

## Mandatory Review Checklist

Every item must be evaluated before the review is considered complete:

| Item | Description |
|------|-------------|
| Executive visual hierarchy | Does the page have a clear focal point? Does importance flow correctly top-to-bottom? |
| Workspace personality | Does this workspace have a distinct identity while clearly belonging to Royaltē OS? |
| Information hierarchy | Is the most important information most prominent? Does secondary content recede correctly? |
| Typography | Are type sizes, weights, and spacing consistent with the OS Design Language? |
| Spacing and alignment | Do all elements feel intentionally placed? Is the grid consistent? |
| Executive Icon System™ | Are all icons sourced from `royalte-icon-registry.svg`? No inline SVGs, no unicode characters? |
| Workspace colour palette | Is the workspace accent color applied correctly and consistently throughout? |
| Ambient module glow | Does the `ws-dept--*` ambient elevation feel subtle but present? |
| Hover behaviour | Do all interactive elements have correct, smooth hover states? |
| Motion consistency | Do count-up animations, ring fills, and transitions match other workspaces? |
| Navigation consistency | Is the active nav item correctly highlighted in the workspace accent color? |
| Card balance | Are all cards visually balanced? No awkward empty space or overflow? |
| Overall polish | Does the workspace look finished — not like a work in progress? |
| Emotional impact | Does this feel premium? Would an artist trust their career data to this interface? |

---

## Quality Standard

The final 2–3% of quality is found by experiencing the product — not by reading code.

Any inconsistency discovered during this review must be corrected before merge approval is requested. No exceptions.

---

## What "One Operating System" Means

When an artist navigates between workspaces, it should feel like moving between departments of the same operating system — not opening different web pages.

Every workspace must immediately feel like it belongs to the Royaltē ecosystem. Nothing should feel disconnected, unfinished, or visually inconsistent.

---

## Enforcement

The Engineering Agent conducting development is responsible for completing the Product Review before opening a PR.

The Board is responsible for the final Board Review before approving the merge.

Neither gate may be skipped.
